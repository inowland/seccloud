# Project Docs

This directory holds internal product, planning, and engineering-governance material.

## Layout

- `spec/`: authoritative product truth
- `planning/`: execution blueprints, logs, retrospectives, and status
- `adr/`: architecture decisions
- `engineering/`: engineering process and testing guides
- `templates/`: recurring document templates

## Rule

This directory is intentionally kept separate from the runnable code and example paths so the repo root stays focused on code, tests, examples, and the main entrypoint docs.
