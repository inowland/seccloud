# M02: Investigation Product

## Objective
Deliver the first analyst workflow for evidence-backed case creation, investigation, and feedback.

## User-Visible Outcome
Analysts can investigate the privileged behavior scenario pack in-product using timelines, evidence bundles, peer comparisons, case dispositions, and a temporary internal demo surface.

## Required Capabilities
- evidence retrieval backed by immutable source pointers
- entity and case timeline assembly
- case creation from detections
- analyst disposition, suppression, and feedback capture
- an internal demo API or UI surface that can render evolving investigation state from projected data
- productized scenario or policy packs for:
  - compromised privileged identity using unfamiliar resources
  - unusual repo clone or export by engineer
  - unusual document or dataset access
  - unusual external sharing
  - benign role, project, or team change as a false-positive control
- an internal API, CLI, or demo UI surface sufficient to execute the workflow during the PoC

## Acceptance Criteria
- analysts can investigate the fixed privileged behavior scenario pack without raw-log spelunking
- most investigations do not require engineering assistance
- evidence and explanations remain stable enough for repeated customer use
- the demo surface can show evolving event, detection, and case state without bypassing the underlying PoC contracts

## Dependencies
- `m01`

## Explicitly Out Of Scope
- production use of advanced representation learning as the main differentiator
- broad prevention or blocking
- agent-first investigation workflows
