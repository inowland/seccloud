# M03: Behavioral Scoring Alpha

## Objective
Add customer-specific behavioral scoring on top of the security data substrate and investigation workflow.

## User-Visible Outcome
The system can prioritize detections using customer-local historical context while still producing understandable reasons and evidence.

## Required Capabilities
- stable feature extraction interface
- model artifact registry inside the customer tenancy
- offline training and evaluation path
- online scoring interface for supported event classes
- explanations tied to structured evidence and reason codes

## Acceptance Criteria
- scoring materially improves prioritization over baseline rules
- seeded scenarios and benign-change scenarios can be evaluated repeatedly
- alerts remain explainable enough for analyst trust

## Dependencies
- `m01`
- `m02`

## Explicitly Out Of Scope
- broad cross-domain model generalization
- cross-customer learning on raw data
- response automation as a primary value surface
